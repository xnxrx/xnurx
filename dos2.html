📖 Technical explainer · UXSS new tab vectors (2026 research)

🕳️ Universal XSS (UXSS) bypasses the same-origin policy entirely, allowing script execution in any domain context.
Modern techniques include: window.open() reference retention, window.name smuggling, postMessage gadget chains, Blob URL origin inheritance, and opener DOM manipulation.

Why "new tab / new window"? Many WebView apps (wallet, banking, chat) handle cross-origin navigation incorrectly. When a new tab retains JavaScript references, an attacker can navigate the tab to a victim domain while keeping script handle → read localStorage, cookies, CSRF tokens.

✅ Mitigation: Use noopener, noreferrer, enable Site Isolation, set Cross-Origin-Opener-Policy: same-origin and upgrade WebView to latest Chromium/WebKit.
