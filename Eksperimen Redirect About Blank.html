Eksperimen Redirect about:blank - Teknik Baru

Membuka tab/window baru dengan teknik redirect tersembunyi

Klik tombol di bawah untuk mencoba berbagai teknik eksperimental yang membuka tab baru dengan about:blank yang kemudian redirect ke URL target. Teknik ini fokus pada variasi karakter setelah ? atau karakter pemisah lain.

Catatan Penting

Beberapa teknik mungkin diblokir oleh browser modern. Untuk hasil terbaik, coba nonaktifkan pemblokir pop-up atau gunakan browser dengan pengaturan keamanan lebih longgar.

Konsep Inti

Inti dari teknik ini adalah membuat URL seperti about:blank?.https://example.com yang saat dibuka di tab baru hanya menampilkan https://example.com di address bar. Kita eksperimen dengan berbagai karakter khusus setelah ? dan pola lain yang belum umum.

Teknik 1: ?. (Standar)
Sudah Diketahui
Menggunakan ?. setelah about:blank. Ini adalah teknik dasar yang sudah diketahui.
about:blank?.https://meta.com
Teknik 2: ?| (Pipe)
Eksperimen
Menggunakan karakter pipe | setelah tanda tanya. Karakter pipe kadang diinterpretasikan berbeda oleh browser.
about:blank?|https://github.com
Teknik 3: ?# (Hash)
Eksperimen
Menggunakan tanda hash # setelah tanda tanya. Tanda hash biasanya untuk anchor dalam URL.
about:blank?#https://stackoverflow.com
Teknik 4: ?! (Bang)
Baru
Menggunakan tanda seru ! setelah tanda tanya. Tanda seru kadang digunakan dalam konteks shebang.
about:blank?!https://news.ycombinator.com
Teknik 5: ?\ (Backslash)
Eksperimen Tinggi
Menggunakan backslash \ setelah tanda tanya. Backslash adalah escape character di banyak sistem.
about:blank?\https://www.wikipedia.org
Teknik 6: ?/ (Slash)
Baru
Menggunakan forward slash / setelah tanda tanya. Biasanya tanda tanya dan slash tidak digabungkan dalam URL.
about:blank?/https://www.google.com
Teknik 7: ?: (Colon)
Eksperimen
Menggunakan titik dua : setelah tanda tanya. Titik dua sudah digunakan dalam skema URL (http:).
about:blank?:https://www.youtube.com
Teknik 8: ?& (Ampersand)
Baru
Menggunakan ampersand & setelah tanda tanya. Ampersand biasanya untuk memisahkan parameter query.
about:blank?&https://www.reddit.com
Teknik 9: ?? (Double Question)
Eksperimental
Menggunakan dua tanda tanya ??. Mungkin diinterpretasikan sebagai parameter query kosong.
about:blank??https://twitter.com
Teknik 10: ?= (Equals)
Baru
Menggunakan tanda sama dengan = setelah tanda tanya. Biasanya untuk memberi nilai pada parameter query.
about:blank?=https://www.netflix.com
Teknik 11: ?→ (Unicode)
Eksperimen Tinggi
Menggunakan karakter Unicode (panah kanan) setelah tanda tanya. Mungkin diinterpretasikan berbeda oleh parser URL.
about:blank?→https://www.amazon.com
Teknik 12: ?[spasi] (Space)
Eksperimen
Menggunakan spasi setelah tanda tanya. Spasi biasanya di-encode sebagai %20 dalam URL.
about:blank? https://www.microsoft.com
