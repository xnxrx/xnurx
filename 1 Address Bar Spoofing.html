📚 Technical Details

VULNERABILITY DETAILS (from Chromium Issue #40091336):
=======================================================
Universal XSS by using "..;@" within the url.

VERSION:
- Chrome Version: 66.0.3359.122 (stable)
- Operating System: iOS

ROOT CAUSE:
When executing "history.replaceState('','','..;@www.google.com:%3443/')" from any domain using HTTPS,
the URL in address bar is replaced to the target domain, but the page content remains under attacker control.

IMPACT:
- Unrestricted XHR requests to the spoofed domain
- Cookie theft from the spoofed domain
- Open redirection on all domains

WORKAROUND (Chrome Team):
- Chrome 67+ crashes when detecting this pattern
- Real fix requires WebKit patch from Apple
