⚠️ Hanya untuk riset & uji coba lokal — halaman ini mendemonstrasikan celah address bar spoofing pada browser macOS yang belum di-patch (Arc, Dia, Safari, Zen). Uji di lingkungan Anda sendiri.

📜 CVE-2025-30467 / CVE-2025-43327 Safari / WebKit

Menggunakan history.pushState() + navigasi tab baru. Pada browser rentan, address bar membeku dengan URL palsu meski halaman sudah berpindah.

// Teknik yang digunakan
window.history.pushState({}, "", "https://www.apple.com/id/");
setTimeout(() => location.href = "https://example.com", 100);
⚡ Klik tombol untuk uji coba

🖥️ CVE-2025-13132 Dia / Arc (fullscreen)

Mode layar penuh tanpa notifikasi + UI palsu. Uji di tab baru/window untuk melihat apakah browser menampilkan toast/banner peringatan.

// Tanpa notifikasi pada versi rentan
document.documentElement.requestFullscreen();
// lalu gambar div hitam meniru bilah alamat
🔲 Klik tombol untuk uji layar penuh

🔗 CVE-2026-44659 Zen / Firefox-based

Subdomain sangat panjang → bilah alamat terpotong, menyembunyikan domain asli penipu.

https://bank-bca.com.very.long.subdomain.aaaaaaaa... [penipu.com]

🕸️ CVE-2025-14812 style Arc / Chromium

Iframe + skema URI (tel://, x-apple:) membingungkan address bar. Uji tab/window baru.

<iframe src="tel://support"></iframe>
setTimeout(() => top.location = 'https://google.com', 80);
🎭 Klik untuk menguji spoofing via iframe/scheme

🧠 Bagaimana cara membaca hasil uji ?

  • ✅ AMAN (PATCHED) — address bar menunjukkan URL yang benar setelah halaman selesai dimuat, atau muncul notifikasi layar penuh.
  • ⚠️ RAWAN (VULNERABLE) — address bar tetap menampilkan URL palsu (contoh: apple.com) meskipun halaman berisi example.com, atau fullscreen tidak menampilkan peringatan toast, atau subdomain panjang memotong domain utama.
  • Untuk real case, Anda bisa menguji di Arc, Dia, Safari, Zen, Orion, Brave versi lawas atau versi terbaru untuk memverifikasi patch.
💡 Tips: Setelah klik tombol "TAB BARU" atau "WINDOW BARU", perhatikan address bar dengan seksama. Apakah URL berubah? Apakah ada UI aneh?
